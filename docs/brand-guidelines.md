# Ultimate Back Office Brand Guidelines

## Purpose

This document defines the official visual design standards for all Ultimate Back Office products.

All applications, websites, dashboards, emails, onboarding flows, and marketing materials should follow these standards.

# Global Design Rules

## Background Color

Use on all pages:

```css
#F5F5F5
```

Never use pure white as the default page background.

White should be reserved for cards, forms, and content containers.

---

## Charcoal Standard

Use:

```css
#222222
```

for:

* Headings
* Navigation
* Icons
* Logos
* Dark buttons
* Primary text

Never use:

```css
#000000
```

---

# Component Standards

## Cards

```css
background: #FFFFFF;
border: 1px solid #E5E5E5;
border-radius: 16px;
box-shadow: 0 2px 6px rgba(0,0,0,.05);
```

---

## Inputs

```css
height: 48px;
border-radius: 10px;
border: 1px solid #CCCCCC;
```

Focus state:

Use active brand color.

---

## Primary Buttons

```css
border-radius: 10px;
padding: 12px 20px;
font-weight: 600;
```

Button color should match the active product brand.

---

## Secondary Buttons

```css
background: #FFFFFF;
border: 1px solid #D9D9D9;
color: #222222;
```

---

## Module Pills

* Active modules

# Layout Standards

All products should use a shared layout structure:

* Header
* Navigation
* Content Area
* Footer

Accounts, Lead Hub, 247SP, EMD, SSP, TUHWD, and future modules should feel like a single ecosystem.

---

# Design Philosophy

The Ultimate Back Office ecosystem should feel:

* Professional
* Modern
* Trustworthy
* Simple
* Approachable
* Easy for non-technical business owners

Avoid:

* Corporate enterprise styling
* Heavy gradients
* Excessive animations
* Dark mode by default
* Overly complex interfaces

The goal is clarity, speed, and usability for local service business owners.
