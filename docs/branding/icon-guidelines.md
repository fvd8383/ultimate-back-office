# Icon Guidelines

Use `app-icon.svg` for module identity in sidebar navigation, app switchers, cards, and compact launch surfaces.

Use `favicon.svg` only for browser tab icons, bookmark icons, and external contexts that specifically request a favicon.

Shared UI icons that are not tied to a product brand should live under `public/assets/ui/icons/`. Shared illustrations belong under `public/assets/ui/illustrations/`, and reusable badge artwork belongs under `public/assets/ui/badges/`.

## Placeholder Policy

Use placeholders only when final brand artwork is not approved. Placeholder assets live under `public/assets/placeholders/` and should be visually neutral.

Social image work should start from each brand folder's `social-placeholder.md`. Approved social preview images should live in the relevant brand folder and be documented there before they are wired into pages.
